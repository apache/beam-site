Apache Beam

The unified programming model for Big Data, running on any engine

Beam provides an advanced unified programming model, allowing you to implement batch and streaming data processing jobs.
Your code runs on any execution engines.

Unified Programming Model

The same programming model for batch and streaming.

Execution Engine Agnostic

The same code runs on any execution engine. Don't rewrite your jobs: they will run on Apache Flink, Apache Spark, ... without any change.

SDKs and IOs

To write your jobs, you have the choice: use the standard Java SDK, use the Python SDK, ... Beam also provides a rich connectivity layer: the IOs to consume or produce data on other systems (Apache Kafka, ...).

Overview
Beam is composed by:

  • An unified programming model providing support of both batch and streaming jobs design.
  • SDKs and DSLs implementing the model: you have the choice of the language you want to use. Pick your: Java, Python, Scala, ...
  • Thanks to the runners, your jobs can run everywhere, on any execution engine: Apache Flink, Apache Spark, Google Cloud Dataflow, ...
  • The IOs allows you to get or put data with other data sources: Apache Kafka, JMS, Apache Cassandra, ...
